装饰器模式

介绍

允许向一个现有的对象添加新的功能，同时又不改变其结构。这种类型的设计模式属于结构型模式，它是作为现有的类的一个包装。

应用场景：

  • 当用于实现横切关注点
  • 数据校验
  • 事务处理（这里的事务类似于数据库事务，意味着要么所有步骤都成功完成，要么事务失败） 缓存
  • 日志
  • 监控
  • 调试
  • 业务规则
  • 压缩
  • 加密
